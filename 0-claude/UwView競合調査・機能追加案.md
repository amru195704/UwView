## 1. 競合調査

### 1.1 背景

UwView の核心は「メモリマップドファイル（mmap）＋スパース行インデックスにより、2億行クラスのテキストファイルをページモードで即時表示し、索引完了後に行モードへ昇格する」という2段階UX。この設計が現存ツールにどこまで存在するかを調査した。

### 1.2 主要競合ツール一覧

| ツール | 対応OS | mmap/即時表示 | 最大規模 | 日本語エンコード自動判定 | クロスプラットフォーム |
|---|---|---|---|---|---|
| **UwView（本作）** | Win / Mac / Linux / Web | ✅ ページモード即時 | 2億行（設計上さらに拡張可） | ✅ UTF-8 / SJIS / EUC-JP 自動 | ✅ Avalonia |
| EmEditor | Windows専用 | ✅ | 1,099億行・16TB | ✅ | ❌ |
| large-text-viewer (Rust OSS) | Win / Mac / Linux | ✅ memmap2 | 100GB超 | ❌ | ✅ |
| glogg | Win / Mac / Linux | △ | 数GB程度 | ❌ | ✅ OSS |
| BigFile | Windows | ✅ | 4GB+ | ❌ | ❌ |
| LogExpert | Windows | ✅ ストリーム | 数GB程度 | ❌ | ❌ |
| LogViewPlus | Windows | △ | 数GB程度 | ❌ | ❌ |
| Large Text File Viewer | Windows | ❌ チャンク読み | 248GB / 21億行 | △ | ❌ |

### 1.3 UwView の差別化ポイント

以下3点を**同時に満たすツールは現時点で存在しない**。

1. **クロスプラットフォーム（Win/Mac/Linux/Web）**  
   Avalonia による全OS統一描画。EmEditor・BigFile・LogExpert はWindows専用。

2. **日本語エンコーディング自動判定（UTF-8 / Shift-JIS / EUC-JP）**  
   日本語市場特有の要件。Rust製OSS含め非対応のものがほとんど。

3. **2億行クラスの超巨大ファイルを索引なしで即時表示（ページモード）→行モード昇格**  
   「待たされない」体験を設計レベルで保証している点が独自。

## 2. 機能追加案

Phase3以降の候補。優先度は競合調査・UX観点から設定。

---

### 🔴 優先度高（コア強化・最大の差別化）

#### ① 文字列検索（背景スレッド・バイト列直接スキャン）

- mmapバイト列をマルチスレッドでスキャンし、エンコーディング変換後に正規表現マッチ。
- 検索結果はバイトオフセットのリストとして保持 → ヒット行へ即ジャンプ。
- 将来的に Boolean 式（`error AND timeout`）まで対応すると本格ログ解析ツールになる。
- **実装ポイント**: 索引構築とは独立したスレッドで動かし、ページモード中でも検索可能にする。

#### ② キーワードハイライト（複数色）

- 複数キーワードを色分けして同時ハイライト。
- 自前描画コントロール（`TextView`）で `DrawText` の部分一致を色替えするだけで実現可能。
- プリセット色パレット（エラー=赤、警告=黄、情報=青など）を用意するとログ解析用途に直結。

#### ③ リアルタイム Tail（ファイル末尾追従）

- `FileSystemWatcher` でファイル更新を検知し、末尾表示中は自動スクロール。
- ログ監視用途で需要が大きい。競合（LogExpert・LogViewPlus）が主要機能として訴求している。
- **実装ポイント**: ファイル末尾に追記される想定で、索引の末尾チェックポイントを動的に延長する。

---

### 🟡 優先度中（使い勝手向上）

#### ④ ブックマーク

- 任意行にブックマークを付け、前後ジャンプ・一覧パネル表示。
- バイトオフセットで保持するのでエンコーディング切替後も有効。
- セッション保存（ファイルパスとオフセットのペアをJSONで保存）まで実装すると再開が楽。

#### ⑤ 行フィルタ表示（仮想フィルタビュー）

- 特定キーワードにマッチした行のバイトオフセットリストを「仮想ドキュメント」として扱い、マッチ行だけを別ビューとして表示。
- 元ファイルは一切変更しない。AND/OR 条件の組み合わせに対応すると差別化になる。
- **実装ポイント**: `LineDocument` の `GetLine` をラップした `FilteredLineDocument` を作るだけで既存設計に乗る。

#### ⑥ ミニマップ（スクロールバー横ヒートマップ）

- ファイル全体を縦に圧縮し、検索ヒット・ブックマークの位置を色点で表示。
- 2億行のどこに何があるかを一目で把握できる視覚ナビゲーション。
- 自前描画コントロールを活かした「UwViewらしい」見せ場になる。
- 検索・ハイライト実装後に追加するのが自然な順序。

---

### 🟢 優先度低（将来拡張）

#### ⑦ 構造化データプレビュー（JSON / CSV / XML）

- 表示モードを「テキスト ↔ 整形済み」で切替。
- 巨大JSONダンプ・CSVの閲覧に有効。BigFile が同様の機能を持つ。

#### ⑧ 行範囲エクスポート

- 指定行範囲のテキストをクリップボードまたは新規ファイルに出力。
- 「2億行のうちここだけ欲しい」という用途。バイトオフセット範囲の `IByteSource.Read` で実装可能。

#### ⑨ 複数ファイルのタブ ＋ タイムスタンプマージビュー

- 複数ログファイルをタブで管理し、タイムスタンプでソートしてマージ表示。
- 本格的なログ解析ツールへの発展形。実装コストは高い。

## 3. 推奨実装順（Phase3）

```
① 文字列検索（背景スレッド）
    ↓
② キーワードハイライト（複数色）＋ ⑥ ミニマップ（同時）
    ↓
③ リアルタイム Tail
    ↓
④ ブックマーク  →  ⑤ 行フィルタビュー
```

①②⑥の3点が揃った時点で「クロスプラットフォーム対応・日本語エンコード自動判定・2億行対応のログ解析ビューア」として明確なポジションが確立される。
