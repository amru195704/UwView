# UwView 販売戦略（海外展開・収益化・多言語化）

作成: 2026-07 ／ 対象: UwView（Avalonia 製・大容量テキストビューア）

## エグゼクティブサマリ

UwView は「巨大テキストを待たされずに開く」性能で明確な差別化を持つ。国内は Vector 作者登録完了・窓の杜へ掲載依頼済みで、次の一手は**海外展開＋収益化**。武器は **OSM 日本全域 51GB・8.9億行の実測**という世界的データセットでの客観的証拠。収益化は OSS 文化に馴染む **Pro版＋GitHub Sponsors** を軸とし、そのために**日英2言語化**を先行整備する。最短ロードマップは「Sponsors 開設 → 英語UI/README → Pro軽量機能 → Pro版リリース → 海外サイト登録 → 企業向けライセンス」。

---

## 1. 現状と課題

- 性能: 巨大テキストビューアとして圧倒的な性能（省メモリ・即時表示）。
- 配布: GitHub Releases に各OS向けビルドを公開済み（Win/Mac/Linux、WASM版あり）。
- 国内: Vector 作者登録完了、窓の杜へ掲載依頼済み。
- 実証: 51GB・8.9億行（OSM 日本全域）を開けることを実測確認、README に結果掲載済み。
- 課題: 海外展開が未着手。UI が日本語のみで、**多言語化（英語）が必要**。

---

## 2. 最大の武器 ── OSM 日本全域 実測（差別化の核）

「最大2億行」は設計上の目安に過ぎず、**実際には8.9億行でも破綻しない**ことを、世界的に有名な OSM データで客観的に示せる。この一点が国内外の掲載審査・技術者コミュニティに強く刺さる。

| 指標 | 実測値 |
|------|--------|
| ファイルサイズ | 51 GB |
| 行数 | 892,239,125 行（`wc -l` と完全一致） |
| ページモード表示 | 12 ms / 3 ms |
| 索引構築 | 172 秒（283 MB/s） |
| 末尾ジャンプ | 0.006 ms |
| 索引サイズ | 26.6 MB |
| メモリ常駐 | 33 MB ＋ 索引 26 MB（ファイル本体は非常駐） |

戦略的意味:
- 「行数よりストレージ容量が先に効く」という実データ特性まで示せる技術的説得力。
- OSM は海外技術者に馴染みが深く、実測ストーリーがそのまま広報素材になる。
- SourceForge / FossHub / 窓の杜などの掲載審査で非常に有利。

## 3. 海外展開の方向性

### 刺さる理由
- 巨大ログ解析の需要が世界的に高い。
- OSM データは海外技術者に馴染みがあり、実測が響く。
- Windows / macOS / Linux のクロスプラットフォーム対応。
- WASM 版が存在し、話題性・デモ導線として強い。

### 登録すべき海外サイト
1. **SourceForge** — OSS 配布の定番。DL 導線と露出。
2. **FossHub** — 高速配布・信頼性の高いミラー。
3. **OSDN** — 日本語 OSS コミュニティ（国内外の橋渡し）。
4. **Product Hunt** — WASM 版デモを軸にローンチ露出（話題性が高い）。

> 注意: 採用ライセンス（PolyForm Internal Use）は OSI オープンソースではないため、OSS前提の SourceForge / FossHub は掲載可否を事前確認すること。フリーウェア/シェアウェア系（Vector・窓の杜）や自サイト・GitHub Releases・Product Hunt は問題なし。

---

## 4. 多言語化（日英2言語）

### 英語対応が必須の理由
- 世界の技術者の大多数が英語圏／英語で情報収集する。
- README 英語版とアプリ UI の一致が信頼性を高める。
- 海外サイトで紹介されやすく、GitHub スター数の伸びに直結。

### 方針
- **日本語＋英語の2言語**に限定（韓国語・中国語は翻訳コスト・文化的負荷・問い合わせ対応の観点から今回は除外）。

### Avalonia での実装方針
- `ResX`（`Strings.ja.resx` / `Strings.en.resx`）で文字列を管理。
- UI テキストをリソースキー化（ハードコード排除）。
- 設定画面に「Language」項目を追加。
- 言語設定は JSON で永続化。
- README の英語版（`README.en.md`）を追加し、本文の実測表も英訳。

## 5. 収益化モデル

### ライセンス（確定）

- **PolyForm Internal Use License 1.0.0**（source-available。ソースは公開するが OSI オープンソースではない＝「公開＋非商用/社内無料」の両立）。
- 無料: 個人利用＋**企業の社内業務利用**。有料: **配布・組込み・転売・OEM・第三者提供**（別途「商用/再配布ライセンス」）。
- 旧 MIT は撤回（商用配布も無償許可のため意図不一致）。詳細は `UwViewライセンス方針.md`。

### 結論：商用(再配布)ライセンス ＋ Pro版 ＋ GitHub Sponsors

根拠:
- 技術者は広告を嫌う（広告モデルは不採用）。
- OSS 文化では寄付・Pro版が受け入れられやすい。
- UwView は巨大ログ解析ツールとして実用価値が高い（対価が正当化される）。
- 海外ユーザーは Sponsors を使う文化が強い。

## 6. UwView Pro版 設計案

### Pro版で提供すると喜ばれる機能
- 検索高速化（マルチスレッド＋SIMD 強化）。
- 検索結果の保存・エクスポート。
- Tail 強化（リアルタイム解析・色分け・通知）。
- 高度なテーマ・フォント設定。
- 索引キャッシュ（次回起動時に即・行モード）。
- 商用利用ライセンス。

|------|------|
| 個人向け | ¥500〜¥1,000 |
| 企業向け | ¥50,000〜¥200,000 / 年（サポート付き） |

## 8. ロードマップ（最短で収益化する流れ）

1. GitHub Sponsors を開設。
2. 英語 UI（`Strings.en.resx`）を追加。
3. README 英語版を作成。
4. Pro版の軽量機能を追加。
5. Pro版をリリース。

7. 企業向けライセンスを用意。

## 9. 実行上の留意点（補足）

- **ライセンス境界**: 本体は PolyForm Internal Use（社内無料・配布は有料）。課金対象は「配布・組込み・転売・OEM・第三者提供」に限られる。Pro機能を足す場合の切り分け（別ビルド／ライセンスキー／別リポ）も先に決めると後戻りが少ない。
- **非OSIの周知**: GitHub 表示は「Other」。OSS前提のディレクトリ（SourceForge/FossHub等）は非OSIを敬遠しうるので、掲載可否を事前確認（Vector・窓の杜・自サイト・GitHub Releases・Product Hunt は問題なし）。
- **多言語の二重保守**: 文言追加時に ja/en を同時更新するルール化（リソースキー未翻訳の検出も）。README も日英の内容ズレに注意。
- **価格は実勢調査を推奨**: 個人¥500〜1,000／企業年額は仮置き。競合（有償ログビューア等）の相場を確認して最終決定。
- **掲載審査の素材化**: OSM 実測表・スクリーンショット・WASM デモURL を「プレスキット」として1か所にまとめておくと、各サイト登録・Product Hunt ローンチが速い。
- **企業向けサポートの線引き**: 年額にどこまでのサポート（応答時間・改修対応）を含むかを明文化。
