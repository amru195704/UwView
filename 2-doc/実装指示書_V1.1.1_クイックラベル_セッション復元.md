# 実装指示書：V1.1.1 — 右クリック着色 ＋ セッション復元/最近使った/お気に入り

## 0. 位置づけ
Ver1.1 で完成した色分けハイライタ・検索・選択系の上に、**日常運用で最も頻度の高い2機能**を足す小刻みリリース。
- **機能1: 選択語の右クリック着色（クイックカラーラベル）** ＝ 完成済みハイライタを「実戦速度」で使う入口。最重要。
- **機能2: セッション復元・最近使ったファイル・お気に入り**（その他4機能 D）＝ 起動ごとの手間を削減。

- **実装先: UVF / UVP 両対応**。ハイライタ同様、コア・永続化・UIは**公開 UwView 側の共有コード**に置き、UVP は再利用する。
- 前提資産（Ver1.1）: マッチャ `CompiledHighlighter`、`AppSettings.Highlighters`、32色パレット、
  管理ダイアログ `HighlighterWindow`、公開 `TextView`。**永続化フィールド（履歴・per-file状態・セッション・recent・favorites）は AppSettings に準備済み**。
- ※クラス名・名前空間は現行実装に合わせる（本書の名称は Ver1.1 報告書準拠。相違あれば現行を優先）。

# 機能1：選択語の右クリック着色（クイックカラーラベル）

## 1-1. 要求仕様（丸山さん決定 2026-07-18）
- 本文で単語/文字列を選択 → **右クリック** → コンテキストに **1〜N の色パレットを簡易表示**
  （＝現在の色セット。管理ダイアログの自動色循環と同じパレット順）→ **色をクリックすると選択語がその色で即着色**
  （ファイル全体の同一文字列が一斉着色）。
- **キーボードショートカット（Ctrl+Shift+1–9）は採用しない**。番号スロット／トグルの概念も廃止（丸山さん指示）。
- パレット末尾に「**解除**（選択語の着色を外す）」と「**管理ダイアログ…**（詳細設定へ）」を併記。
- 既定マッチ: **literal・大小区別**（付与後に管理ダイアログで個別変更可）。
- 登録規則は Ver1.1 のハイライタ実体（`AppSettings.Highlighters`）と**同一**。
  永続化・管理ダイアログでの編集・プリセット保存・エクスポート(.uwvhl)の対象になる。

## 1-2. 前提：本文のサブ行テキスト選択（TextView 拡張）★最初にやる
現状 UVF の本文選択は**行選択**（Ver1.1）。着色対象を得るには**行内の文字範囲（サブ行）選択**が要る。
- 公開 `TextView` に**線形の文字範囲選択**を追加（ドラッグで文字単位に範囲選択、`SelectedText` を公開）。論理座標（行番号×文字index）で保持。
- 既存の行選択・矩形選択（UVP）・ブックマークトグル等と競合しないよう入力を整理
  （矩形指示書 §3 の「押下→移動閾値でドラッグ種別を確定」方式を踏襲。ダブルクリックで単語選択）。
- **右クリック時に選択が空ならキャレット位置の単語**（word boundary で切り出し）を対象にする。

## 1-3. 右クリックパレットUI（本機能の中心）
- 選択（またはキャレット単語）がある状態で右クリック → コンテキストメニュー上部に**色スウォッチの横並び（1〜N）**を簡易表示。
  - **N＝ハイライタパレット先頭N色（既定10・自動循環と同一。設定で調整可）**。各スウォッチにホバーで番号/色名。
  - すでに選択語に色が付いている場合は、その色に**チェック/枠**を出す。
  - 末尾に「**解除**」「**管理ダイアログ…**」。
- 色クリック → 選択語 T を **literal・大小区別**で color=選択色 の規則として add（既存なら色を更新＝1語1規則）。**可視行再評価で即反映**。
- 空白のみ・空選択（単語も取れない）→ 無効（規則を作らない）。

## 1-4. 動作詳細
- 同じ語に別の色を選ぶ → その語の色を更新（番号スロットは無いので「置換」の概念は不要、常に1語=1規則）。
- 「解除」→ その語の規則を削除。
- 一括クリアは管理ダイアログ側の既存機能（プリセット「なし（全てクリア）」等）で足りる（クイック専用の一括クリアは作らない）。
- 反映は**可視行の再評価のみ**（Ver1.1 の描画方針を流用、巨大ファイルでも影響なし）。検索の黄ハイライト最優先の描画順は不変。

## 1-5. UI 統合
- 右クリックメニューへ色パレットを追加（MainView と UVP の ProMainView・共通実装、UVF/UVP）。en/ja のラベル追加。
- 付与時にステータスバーへ軽い通知（例: `"req-8f2a" を着色（12件）`）。ヒット0でも登録はする（後続ログで着く）。
- 管理ダイアログを開くと右クリック着色分も規則一覧に見え、編集・色変更・セット保存ができる（実体共通のため自動）。

# 機能2：セッション復元・最近使ったファイル・お気に入り（その他4機能 D）

## 2-1. 要求仕様
- **最近使ったファイル（Recent/MRU）**: 開いた順に保持（上限 **N=15**）。メニュー/スタート画面から再オープン。
- **お気に入り（Favorites）**: ユーザーがピン留めした常用ファイル。手動 追加/削除。**並べ替えUIは無し＝追加順表示**。
- **セッション復元**: 起動時に**「前回のファイルを復元しますか？」の確認ダイアログを出し**、OKなら前回のタブ群を再オープンしアクティブタブを復元。

> スコープは **D（セッション＝どのファイル群を開くか＋recent＋favorites）＋前回スクロール位置の軽い復元**。
> per-file のフル状態（ブックマーク・エンコード選択・Follow）の復元は **C（別指示書）**。

## 2-2. 永続化（AppSettings）
準備済みフィールドを実装で埋める。想定スキーマ（現行名に合わせる）:
- `RecentFiles: List<RecentEntry>`（`Path, LastOpenedUtc`。上限15・先頭詰め・重複排除・存在しないパスは読込時に間引く）
- `Favorites: List<string>`（絶対パス。追加順保持・重複排除。並べ替えUIなし）
- `LastSession: SessionState { List<OpenDoc> Docs; int ActiveIndex }`
  - `OpenDoc { Path; long LastTopLine }` … **LastTopLine＝前回の表示先頭行（軽いスクロール位置復元に使う）**。
    エンコード/Follow/マークは C 送り。
- 保存タイミング: 開いた/閉じた/タブ切替/スクロール停止時に Recent・LastSession を更新、アプリ終了時に確定保存。
  Favorites は操作時に即保存。破損に備え原子的書き込み（temp→rename）。

## 2-3. 動作詳細
- **通常起動**: `LastSession.Docs` が空でなければ**確認ダイアログ**「前回のファイル（n件）を復元しますか？」を表示。
  - OK → 順に開き、各ファイルを **LastTopLine 付近までスクロール**して表示。存在しないファイルはスキップしまとめて通知（例: `3件中1件が見つかりませんでした`）。復元後 ActiveIndex を選択。
  - キャンセル → まっさら起動（LastSession は保持し、次回また尋ねる）。
  - 「今後尋ねない／常に復元」等のチェックは任意（付ける場合は設定に保存）。
- **ファイル指定起動（ダブルクリック / D&D / `uwview app.log`）**: **指定ファイルのみを開く。確認ダイアログは出さず、セッション復元もしない**（丸山さん決定）。
  - 即時オープン（Ver1.1 のページモード）で先頭表示、索引はバックグラウンド。
- **Recent**: 開くたび先頭へ（上限15・重複排除）。メニュー「最近使ったファイル」＋「リストをクリア」。欠損は淡色＋クリック時通知し間引く。
- **Favorites**: タブ右クリック/メニューから「お気に入りに追加/解除」。スタート画面・メニューに追加順で常設表示。

## 2-4. UI 統合
- **スタート画面（ファイル未オープン時）**: 「最近使ったファイル」「お気に入り」の2リスト＋「開く」。UVF/UVP 共通。
- **メニュー**: ファイル > 最近使ったファイル（サブメニュー）／お気に入り。
- en/ja リソースにラベル追加。UVP も同UI（配色のみ Pro）。

# 共通事項

## 3. 決定事項（2026-07-18 丸山さん確定）
1. ✅ 着色の起動＝**選択→右クリック→色パレット（1〜N）から選ぶ**。Ctrl+Shift+1–9・番号スロット・トグルは**廃止**。
2. ✅ 着色のマッチ既定＝**literal・大小区別**（ダイアログで個別変更可）。
3. ✅ セッション復元＝**起動時に確認ダイアログ**（OKで復元）。
4. ✅ ファイル指定起動＝**指定ファイルのみ・確認なし・復元しない**。
5. ✅ Recent 上限＝**15件**。
6. ✅ Favorites 並べ替えUI＝**無し（追加順）**。
7. ✅ **前回スクロール位置の軽い復元を入れる**（マーク・Follow・エンコード等のフル状態復元は将来の C）。

### 残る確認（実装時）
- 右クリックパレットに出す **N（既定10）** でよいか、スウォッチに番号/色名ラベルを出すか。
- 「解除」「管理ダイアログ…」の文言（en/ja）。
- セッション復元ダイアログに「今後尋ねない」を付けるか。

## 4. 受入基準
1. 語を選択→右クリック→色パレットから色を選ぶと、全同一語が即着色。別の色を選ぶと更新、「解除」で外れる。
2. 空選択で右クリック → キャレット位置の単語が対象。巨大ファイルでも即着色（可視行再評価・カクつき無し）。
3. 右クリック着色分が管理ダイアログの規則一覧に現れ、色変更・セット保存・.uwvhl エクスポートの対象になる。
4. 通常起動で復元確認ダイアログが出て、OKで前回タブ群＋前回スクロール位置付近が復元。欠損はスキップし件数通知。キャンセルでまっさら。
5. ファイル指定起動（ダブルクリック/D&D/CLI）は、そのファイルのみを開き、確認も復元もしない。
6. Recent が新しい順・上限15・重複なし・欠損間引き。Favorites の追加/解除が再起動後も保持（追加順）。
7. UVF・UVP 双方でビルド成功・既存機能（検索/ハイライタ/矩形(UVP)/Tail/ミニマップ）と干渉なし。単体テスト全緑。

## 5. 実装手順
1. **TextView 線形サブ行選択**（公開 UwView）＋ `SelectedText`／単語切り出しユーティリティ。ユニットテスト（全角/タブ/行端/単語境界）。
2. **右クリック色パレット配線**：選択（or キャレット単語）→ 右クリックメニューに色スウォッチ(1〜N)＋解除＋ダイアログ → 色選択で `AppSettings.Highlighters` に literal・大小区別の規則を add/更新/削除 → 可視行再評価。en/ja・ステータス通知。テスト。
3. **AppSettings 永続化**：RecentFiles(15)/Favorites/LastSession(LastTopLine含む) の読み書き（上限・重複排除・欠損間引き・原子的書き込み）。テスト。
4. **セッション復元**：起動シーケンスに確認ダイアログ＋復元（スクロール位置復元・欠損スキップ通知）、ファイル指定起動の分岐（復元しない）。
5. **UI**：スタート画面（Recent/Favorites）＋ファイルメニュー。UVF/UVP。
6. **UVP リビルド**で同UIを確認。受入基準の通し確認。バージョンを **1.1.1** に更新（About 表示）。
