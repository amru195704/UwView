# 色分けハイライタ ウィンドウ 右端見切れ修正メモ（Code向け・2026-07-18）

## 問題
「色分けハイライタ」ウィンドウで、右端「背景色」の▲▼ステッパーが見切れて操作できない。

## 原因
1行の列幅合計がウィンドウ幅を超えている。かつ右端にトレーリング余白が無く、最後のコントロールが
窓の縁に貼り付いて切れている。「パターン」入力欄が横幅を食い過ぎているのも一因。

## 対策（優先順）
1. **可変幅は「パターン」列だけにする**：パターンを `Width="*"`、その右の列（正規表現・Aa・行全体・
   文字色・背景色・▲▼）はすべて**固定幅**にする。→ 窓を縮めても縮むのはパターン欄だけ、右端は常に表示。
2. **右端にトレーリングのスペーサーを置く**（＝丸山さん案の「ダミー空白」）：
   末尾に `ColumnDefinition Width="12〜16"` を追加、または行コンテナに `Margin="0,0,16,0"`。
   最後の▲▼が縁に触れなくなる。
3. **ウィンドウに `MinWidth` を設定**：固定列合計＋パターン最小幅。これ以上は縮められないようにする。
4. **行全体を横 `ScrollViewer` で包む**（保険）：狭くされても右端に到達可能に。
   ヘッダー行とデータ行は同じ列定義／`SharedSizeGroup` で幅を揃える（DataGrid なら列幅を共有）。
5. 細部の横幅捻出：文字色/背景色の `#000000` 入力欄の幅を詰める、▲▼ステッパーを小型化する。

## 実装メモ（Avalonia）
- 行が Grid の場合、各行を別々に組むと列がズレるので `Grid.IsSharedSizeScope` ＋ 各列 `SharedSizeGroup`、
  もしくは `DataGrid`（末尾列を固定幅＋右パディング）で統一する。
- ヘッダーの「背景色」ラベルと本体の背景色列の右端がズレていないかも合わせて確認。
