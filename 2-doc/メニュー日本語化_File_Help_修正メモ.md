# UwView メニュー「File / Help」が日本語化されない — 修正メモ（Code向け・2026-07-20）

## 症状
アプリの言語を**日本語**に切り替えても、メニューの「**File**」「**Help**」が英語のまま。
→「**ファイル**」「**ヘルプ**」に表示されるようにする。（丸山さん報告）

## 原因（推定）
1. トップレベルメニューの `Header` が**ローカライズ資源にバインドされずハードコード**（`Header="File"` / `Header="Help"`）、
   または該当の **ja リソースキーが未登録**。
2. macOS の場合、**NativeMenu（macOS標準のアプリメニューバー）**が in-window Menu と別定義で、言語切替に追随していない可能性。
   （Windows/Linux は in-window Menu、macOS は NativeMenuBar と、実体が分かれているケースに注意）

## 修正
- File/Help を含むメニューの `Header` を、既存の **en/ja ローカライズ資源にバインド**し、ja に
  「ファイル」「ヘルプ」を追加（Ver1.1で en/ja リソースは導入済みなので、その仕組みに載せる）。
- **Windows/Linux の in-window `Menu` と、macOS の `NativeMenuBar` の両方**を確認・修正。どちらか片方だけ英語が残らないように。
- 言語切替の反映タイミング（即時／次回起動）を確認。即時反映が難しければ「変更は再起動後に反映」でも可（挙動を統一）。
- ついでに **他の未翻訳メニュー項目**（Edit/View や 開く・検索・設定 等）が残っていないか一覧で総点検し、まとめてja化。

## 受入基準
- 日本語設定でメニューが「ファイル」「ヘルプ」（および配下項目も日本語）で表示される。
- 英語設定では従来どおり "File" / "Help"。
- **UVF / UVP 両方**・Windows/macOS/Linux で確認（特に macOS のネイティブメニュー）。
