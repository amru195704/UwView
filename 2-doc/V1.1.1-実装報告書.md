## 2. 機能1：選択語の右クリック着色（クイックカラーラベル）

完成済みハイライタを「実戦速度」で使う入口。**選択→右クリック→色パレット** で着色する。

### 語の選択（TextView 拡張・3段階ダブルクリック）

- **ダブルクリックの段階選択**（丸山さん指示）:
  1回目 = **最短の語**（英数字＋アンダースコアの塊。`req-8f2a` の `req` だけ等）
  → 同じ場所で2回目 = **区切りの良いトークン**（`req-8f2a` 全体等）
  → 3回目 = **解除**。別の語をダブルクリックすると最短からやり直す。
- **右クリック時に選択が空ならキャレット位置の語**（トークン）を対象にする。
- 選択語はハイライト表示。語切り出し・表示セル計算は Core `TextCells`（半角=1/全角=2/タブ=8ストップ、
  最短語 `WordAtNarrow`・トークン `WordAt`）に切り出し、**単体テスト7件**追加。

### 右クリック色パレット

- 選択（orキャレット語）で右クリック → **先頭10色のスウォッチ ＋「解除」＋「管理ダイアログ…」** のポップアップ。
- 色クリック → その語を **literal・大小区別** の規則として `AppSettings.Highlighters` に add／更新
  （**1語=1規則**・別の色で更新）。**可視行再評価で全同一語が即着色**。
- 「解除」= その語の規則を削除。「管理ダイアログ…」= ハイライタ管理を開く。
- 付与時にステータスバーへ一時通知（`「word」を着色`）。着色分は管理ダイアログの規則一覧に現れ、
  色変更・セット保存・`.uwvhl` エクスポートの対象になる（実体共通のため自動）。
- 検索の黄ハイライト最優先の描画順は不変。UVP は矩形モード中は語ターゲティングを無効化。

対応: **UVF / UVP 両対応**

## 3. 機能2：セッション復元・最近使ったファイル・お気に入り

### 動作

- **セッション復元**: 通常起動時に「前回のファイル（n件）を復元しますか？」の**確認ダイアログ**。
  OK で前回タブ群を再オープンし、**前回スクロール位置付近**へ復元、アクティブタブを選択。
  欠損はスキップしまとめて通知。キャンセルでまっさら（LastSession は保持し次回また尋ねる）。
- **ファイル指定起動**（ダブルクリック/D&D/CLI 引数）: **指定ファイルのみ・確認なし・復元しない**。
- **最近使ったファイル**（上限15・新しい順・重複排除・欠損間引き）。
- **お気に入り**（★トグルで追加/解除・追加順・重複排除）。
- **スタート画面**（ファイル未オープン時）: 「最近使ったファイル」「お気に入り」の2リスト＋「開く」。
  ダブルクリックで開く。UVF/UVP 共通（配色のみ Pro）。

### 永続化（AppSettings）

- `RecentFiles: List<RecentEntry>`（Path＋最終オープン時刻・上限15）
- `Favorites: List<string>`（絶対パス・追加順）
- `LastSession: SessionState { List<OpenDoc> Docs; int ActiveIndex }`
  （`OpenDoc.LastTopLine` ＝ 前回表示先頭行）
- 破損に備え**原子的書き込み**（temp→rename）。終了時に確定保存。
- 共有 ViewModel は `Services.AppSettingsRef.Current` 経由で参照（App／ProApp が起動時に設定）。

対応: **UVF / UVP 両対応**

## 6. V1.1.1 追補（対話レビューでの改善・確定）

指示書ベースの実装後、実機レビューで以下を追加・修正した（すべて UVF/UVP 両対応・ビルド/テスト緑）。

### タブ別ハイライタ
- 各タブが自分のハイライタを持つ。**タブ→[色名] ／ 共通テーブル[色名]→セット** の形。
  同一名称なら同一色設定を共有、別名なら独立。タブ切替でそのタブのハイライタを適用。
- 実装: `DocumentTabViewModel.HighlighterSetId`。ダイアログはアクティブタブのセットを編集し、
  確定時にそのタブへ割り当てる。クイック着色も現在のタブのセットに追加。

### 色設定ダイアログの操作系
- ボタンを **保存 / 設定 / キャンセル** の3つに再構成。
  - **保存**: 名前を指定して保存＆閉じる（既定＝現在のプリセット名。別名なら別ハイライタ）。
  - **設定**: 名前を尋ねず、その名前のセットへ確定して閉じる（同名の無駄な保存を回避）。
  - **キャンセル / ×**: 編集を破棄して以前の設定に戻して閉じる。編集中はプレビューのみ。
- **プリセット選択＝「そのハイライタへ切替」**（規則の追加ではなく置換）。保存/設定は**名前で確定**。
- **「（なし）」= 独立した空のハイライタ**（全「なし」タブで共有）。他タブの色設定に波及しない。

### クイック着色（右クリック）
- ダブルクリックの語選択を **3段階**（1回目=最短語 → 2回目=トークン → 3回目=解除）。
  段階判定は「クリックが現在の選択範囲内か」で行う。範囲外は新しい語で最短からやり直し。
- 右クリック色パレットの**先頭に「コピー」**を追加（着色か copy か迷わない）。

### 同梱プリセットの拡充（作者ドメイン）
- **GeoJSON**（JSON キー: ジオメトリ種別・coordinates・properties 等）
- **KML**（XML タグ: Point/LineString/Polygon・Placemark・Style 等）
- 既存: 汎用ログレベル / syslog / Web access / JSON ログ / GNSS NMEA。**計7プリセット**。
